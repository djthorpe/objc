1 # objc runtime
2 
3 This is a minimal Objective C runtime written in C, designed to be portable across different platforms,
4 including ARM and x86 architectures, mostly targeting embedded systems. It uses the "gcc" ABI for the
5 moment, as that is the most portable across different platforms.

11 ## Requirements
12 
13 You will minimally need the following tools to build the runtime:
14 
15 - `make` and `cmake` - for the build system
16 - `clang` or `gcc` - for compiling the runtime (`clang` is not supported on Apple Silicon). If you want
17  to cross-compile for ARM, you will need the ARM LLVM toolchain (see below).
18 
19 ### ARM Toolchain
20 
21 For cross-compilation for embedded systems based on some ARM variant, get the ARM LLVM toolchain:\
22 <https://github.com/ARM-software/LLVM-embedded-toolchain-for-Arm/releases>
23 
24 Install this to the `/opt` directory.
25 
26 ## Building the runtime
27 
28 Download the source code from GitHub:
29 
30 ```bash
31 git clone git@github.com:djthorpe/objc.git
32 cd objc
33 ```
34 
35 The `TOOLCHAIN_PATH` environment variable should point to the directory where the toolchain is installed. For example,
36 
37 ```bash
38 # Compile with GCC 15 for MacOS
39 brew install gcc@15
40 TOOLCHAIN_PATH=/opt/homebrew CC=gcc-15 make
41 ```
42 
43 Use the `make tests` target to run the unit .tests
44 
45 For the RP2040 Pico board, you can use the `clang` compiler with the ARM toolchain. The `TARGET` environment variable should be set to the target architecture, such as `armv6m-none-eabi` for the RP2040 Pico board:
46 
47 ```bash
48 # Compile for the RP2040 Pico board
49 CC=clang TARGET=armv6m-none-eabi TOOLCHAIN_PATH=/opt/LLVM-ET-Arm-19.1.5-Darwin-universal make
50 ```
51 
52 You can use the environment variable `RELEASE=1` to build a release version of the runtime, which will
53 enable optimizations and disable debug symbols.
